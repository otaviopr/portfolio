Formulário de Registro de Hóspedes

Ferramentas

Figma, Figjam, Maze, paper sketches

 

Atividades

Ideation, User Flow, UX/UI, Prototyping and user testing

A Zoox Smart Data é uma startup especializada em soluções de inteligência de dados. Seu produto para o setor de hospitalidade, sem atualizações há mais de cinco anos, precisava de uma grande reformulação para atender a pedidos de clientes por novos recursos e uma melhor experiência do usuário. Outro objetivo foi reduzir o tempo necessário para completar o formulário.

Como Lead UX Designer, meu foco foi identificar lacunas no produto, propor inovações em UI/UX e colaborar com o time de Front-end para lançar uma versão completamente redesenhada.

Organização do escopo e etapas

Auxiliei o PO e stakeholders a definir abordagens, etapas e métricas para medir o sucesso, usando o processo de double diamond para alcançar os objetivos.

 

Discovery

Entrevistamos supervisores e gerentes de hotéis para identificar problemas e necessidades. O desafio central foi criar uma nova versão mais clara, usável e alinhada à identidade visual dos hotéis.

Design da Solução

Para o design da solução, foi essencial focar nos principais pontos problemáticos, especialmente no aspecto mobile.

Além disso, tornou-se importante criar um sistema de design básico e personalizável que os clientes pudessem editar facilmente para incorporar sua identidade visual.

Ao criar componentes simples, mas altamente personalizáveis, do sistema de design, proporcionamos aos usuários maior controle sobre como os elementos visuais do produto se alinham à identidade da marca deles.

Foi criada uma tela inicial para explicar o objetivo principal do produto e fornecer dicas que ajudam os usuários a agilizar o processo, tornando-o mais rápido e fácil.

Além disso, um botão secundário foi adicionado para oferecer informações sobre o “FNRH” e explicar por que o hotel precisa das informações dos usuários.

Na tela principal, os usuários podem visualizar todos os hóspedes registrados para as reservas do hotel. Foi essencial implementar um sistema de notificações para crianças menores de idade em reservas, especificamente para informar os usuários sobre leis e exigências relacionadas a hóspedes que viajam com crianças.

No topo da tela, os usuários têm acesso constante aos detalhes de suas reservas. Como muitos usuários podem ter múltiplas reservas, essa funcionalidade fornece uma maneira rápida de visualizar informações relevantes.

Para o fluxo da tela de upload, decidimos criar uma tela de “guia rápido” para explicar e fornecer dicas sobre como tirar melhores fotos dos documentos dos usuários.

Testando

Usando o Maze, compartilhei o protótipo do Figma com vários usuários para testes de tela. O objetivo principal era avaliar a interface e também os novos fluxos introduzidos nesta versão do produto.

Dividi o teste em quatro etapas, representando os principais fluxos do produto.

Os resultados foram impressionantes. Confirmamos a eficácia do novo UX e obtivemos insights mais claros sobre alguns dos novos fluxos implementados.

Em uma escala de “Muito Ruim” a “Muito Bom”, recebemos uma pontuação de 4,7 de um total de 35 respostas.

Ao analisar os fluxos das telas para cada um dos quatro segmentos, descobrimos que o segmento de “pagamento” estava excessivamente confuso e difícil de entender para os usuários.

Como resultado da fase de testes, apresentei a análise aos stakeholders e ao Product Owner (PO) do projeto, e decidimos fazer mudanças no fluxo da tela de pagamento para simplificá-lo.

Delivery

Criar uma documentação detalhada para a equipe de desenvolvimento é uma parte crucial do meu processo.


Neste projeto, a documentação criada no Figma ajudou os desenvolvedores front-end a entender as regras de negócios definidas e auxiliou o Product Owner (PO) na criação das user stories para os desenvolvedores.


Além disso, várias regras e definições front-end foram incluídas no arquivo do Figma para facilitar a consulta pela equipe de desenvolvimento.

Resultados

Alcançamos resultados excepcionais com este projeto. Vários clientes importantes expressaram satisfação, principalmente pela redução no tempo gasto pelos usuários para preencher formulários e pelas novas opções de personalização.


Implementamos uma modal de avaliação ao final de cada processo, e as respostas foram bastante positivas.


Além disso, ao analisar os dados do Google Analytics, observamos um aumento de 25% nas taxas de conclusão de formulários em comparação com antes.
